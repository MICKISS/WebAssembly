¿Que es Web-Assembly?

Web Assembly es un nuevo tipo de código que puede ser ejecutado en navegadores modernos, es un lenguaje de bajo nivel, similar al lenguaje ensamblador, con un formato binario compacto que se ejecuta con rendimiento casi nativo y provee un objetivo de compilación para lenguajes como C/C++ y Rust que les permite correr en la web. También es conocido en la web es una nueva tecnología del navegador que promete revolucionar el desarrollo para el cliente.

Objetivos de Web Assembly

Web Assembly se está creando como un estándar abierto dentro del W3C WebAssembly Community Group con los siguientes objetivos:

Ser rápido, eficiente y portátil: el código de WebAssembly se puede ejecutar a una velocidad casi nativa en diferentes plataformas aprovechando las capacidades de hardware comunes .

Ser legible y depurable: WebAssembly es un lenguaje ensamblador de bajo nivel, pero tiene un formato de texto legible por humanos (cuya especificación aún se está finalizando) que permite que el código se escriba, vea y depure a mano.

Mantener seguro: WebAssembly está especificado para ejecutarse en un entorno de ejecución seguro y aislado. Al igual que cualquier otro código web, hará cumplir las políticas de permisos y del mismo origen del navegador.

No rompa la web: WebAssembly está diseñado para que funcione bien con otras tecnologías web y mantenga la compatibilidad con versiones anteriores.

¿Cómo encaja WebAssembly en la plataforma web?

Una máquina virtual (VM) que ejecuta el código de la aplicación web, por ejemplo, el código JavaScript que alimenta sus aplicaciones.

Un conjunto de API web que la aplicación web puede llamar para controlar la funcionalidad del navegador / dispositivo web y hacer que las cosas sucedan ( DOM , CSSOM , WebGL , IndexedDB , API de audio web , etc.).

¿Por qué utilizar Web Assembly?

Históricamente, la VM solo ha podido cargar JavaScript. Esto ha funcionado bien para nosotros, ya que JavaScript es lo suficientemente poderoso como para resolver la mayoría de los problemas que la gente tiene en la Web hoy en día. Sin embargo, nos hemos encontrado con problemas de rendimiento cuando intentamos usar JavaScript para casos de uso más intensivos como juegos 3D, realidad virtual y aumentada, visión por computadora, edición de imágenes / videos y una serie de otros dominios que exigen un rendimiento nativo.

Puntos fuertes de JavaScript y Web-Assembly

JavaScript es un lenguaje de alto nivel, lo suficientemente flexible y expresivo para escribir aplicaciones web.

WebAssembly es un lenguaje ensamblador de bajo nivel con un formato binario compacto que se ejecuta con un rendimiento casi nativo y proporciona lenguajes con modelos de memoria de bajo nivel como C ++ y Rust con un destino de compilación para que puedan ejecutarse en la web.

Conceptos claves de WebAssembly

Módulo : representa un binario de WebAssembly que ha sido compilado por el navegador en un código de máquina ejecutable. Un módulo no tiene estado y, por lo tanto, como un Blob , se puede compartir explícitamente entre Windows y trabajadores (vía postMessage()). Un módulo declara importaciones y exportaciones como un módulo ES2015.

Memoria : un ArrayBuffer de tamaño variable que contiene la matriz lineal de bytes leídos y escritos por las instrucciones de acceso a memoria de bajo nivel de WebAssembly.

Tabla : Una matriz de referencias de tipo redimensionable (por ejemplo, a funciones) que de otro modo no podrían almacenarse como bytes sin procesar en la memoria (por razones de seguridad y portabilidad).

Instancia : un módulo emparejado con todo el estado que usa en tiempo de ejecución, incluida una memoria, una tabla y un conjunto de valores importados. Una instancia es como un módulo ES2015 que se ha cargado en un global particular con un conjunto particular de importaciones.
